Properties

config

This items configuration in its current state

type

The type of the item. Can be row, column, stack, component or root

contentItems

An array of items that are children of this item

parent

The item that is this item's parent (or null if the item is root)

id

A String or array of identifiers if provided in the configuration

isInitialised

True if the item had been initialised

isMaximised

True if the item is maximised

isRoot

True if the item is the layout's root item

isRow

True if the item is a row

isColumn

True if the item is a column

isStack

True if the item is a stack

isComponent

True if the item is a component

layoutManager

A reference to the layoutManager that controls this item

element

The item's outer element

childElementContainer

The item's inner element. Can be the same as the outer element.

Events

stateChanged

Fired whenever something happened to this item or its children that changes the state of the layout (as returned by layout.toConfig)

titleChanged

Updated whenever the item's title changes as a result of calling setTitle() on either the item or from a container

itemDestroyed

Fired whenever this item or one of its children gets destroyed

itemCreated

Fired whenever an item gets created as child of this item

componentCreated

Fired whenever a component gets created as child of this item

rowCreated

Fired whenever a row gets created as child of this item

columnCreated

Fired whenever a column gets created as child of this item

stackCreated

Fired whenever a stack gets created as child of this item

addChild( itemOrItemConfig, index )

argumenttypeoptionaldefaultdescription
itemOrItemConfigContentItem or Configurationfalse-A content item (or tree of content items) or an ItemConfiguration to create the item from
indexNumbertruelast indexAn optional index that determines at which position the new item should be added.

Adds an item as a child to this item. If the item is already a part of a layout it will be removed from its original position before adding it to this item.

removeChild( contentItem, keepChild )

argumenttypeoptionaldefaultdescription
contentItemContentItemfalse-The contentItem that should be removed
keepChildBooleantruefalseIf true the item won't be destroyed. (Use cautiosly, if the item isn't destroyed it's up to you to destroy it later)

Destroys the item and all it's children.

replaceChild( oldChild, newChild )

argumenttypeoptionaldefaultdescription
oldChildContentItemfalse-The contentItem that should be removed
newChildContentItem or Configurationfalse-A content item (or tree of content items) or an ItemConfiguration to create the item from

Replaces oldChild with newChild

setSize()

Updates the items size. To actually assign a new size from within a component, use container.setSize( width, height )

setTitle( title )

argumenttypeoptionaldefaultdescription
titleStringfalse-the new title

Sets the item's title to the provided value. Triggers titleChanged and stateChanged events

callDownwards(functionName, functionArguments, bottomUp, skipSelf)

argumenttypeoptionaldefaultdescription
functionNameStringfalse-The name of the method to invoke
functionArgumentsArraytrue-An array of arguments to pass to every function
bottomUpBooleantruefalseIf true, the method is invoked on the lowest parts of the tree first and then bubbles upwards
skipSelfBooleantruefalseIf true, the method will only be invoked on the item's children, but not on the item itself

A powerful, yet admittedly confusing method to recursively call methods on item in a tree. Usually you wouldn't need to use it directly, but it's used internally to setSizes, destroy parts of the item tree etc.

emitBubblingEvent( name )

argumenttypeoptionaldefaultdescription
titleStringfalse-the name of the event

Emits an event that bubbles up the item tree until it reaches the root element (and after a delay the layout manager). Useful e.g. for indicating state changes.

remove()

Convenience method for item.parent.removeChild( item )

popout()

Removes the item from its current position in the layout and opens it in a window

toggleMaximise()

Maximises the item or minimises it if it's already maximised

select()

Selects the item. Only relevant if settings.selectionEnabled is set to true

deselect()

Unselects the item. Only relevant if settings.selectionEnabled is set to true

hasId(id)

argumenttypeoptionaldefaultdescription
idStringfalse-An id to check for

Returns true if the item has the specified id or false if not

addId(id)

argumenttypeoptionaldefaultdescription
idStringfalse-The id to be added

Adds an id to an item or does nothing if the id is already present

removeId(id)

argumenttypeoptionaldefaultdescription
idStringfalse-The id to be removed

Removes an id from an item or throws an error if the id couldn't be found

getItemsByFilter(filterFunction)

argumenttypeoptionaldefaultdescription
filterFunctionFunctionfalse-A function that determines whether an item matches certain criteria

Calls filterFunction recursively for every item in the tree. If the function returns true the item is added to the resulting array

getItemsById( id )

argumenttypeoptionaldefaultdescription
idStringfalse-An id specified in the itemConfig

Returns all items with the specified id.

getItemsByType( type )

argumenttypeoptionaldefaultdescription
typeStringfalse-'row', 'column', 'stack', 'component' or 'root'

Returns all items with the specified type

getComponentsByName( componentName )

argumenttypeoptionaldefaultdescription
typeStringfalse-a componentName as specified in the itemConfig

Returns all instances of the component with the specified componentName
