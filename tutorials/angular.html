Using GoldenLayout with Angular

Angular’s popularity is exploding and it’s easy to see why: It provides a set of solutions to common problems and good integration-points for usage of third part components.

Using Angular within third party components (e.g. GoldenLayout) however can be challenging. But fear not, this tutorial is here to help.

The main challenges are

  • working with multiple modules
  • bootstrapping a module directly into a DOM element at runtime
  • injecting external dependencies (container, state) into a module
  • communicating between modules

To address these we’ll build a user management app consisting of two modules: a list of users and a detail panel.

  • Each module will live in its own GoldenLayout container
  • Selecting a user in the list will show its details in the detail panel (communication between modules)
  • The initially selected user will be configured as part of GoldenLayout’s component state (external dependency).

Here’s the result

The user list

angular.module('userlist', [] )
.controller('userlistController', 
    [ '$scope', '$timeout', 'eventHub', 'state', 
    function( $scope, $timeout, eventHub, state ) {

    // Reference to unselect user when another is selected
    var selectedUser = {};

    $scope.users = [
        { name: 'Jackson Turner', street: '217 Tawny End', img: 'men/1.jpg' },
        { name: 'Megan Perry', street: '77 Burning Ramp', img: 'women/1.jpg' },
        { name: 'Ryan Harris', street: '12 Hazy Apple Route', img: 'men/2.jpg' },
        { name: 'Lisa Edwards', street: '33 Maple Drive', img: 'women/2.jpg' },
        { name: 'Noah Jenkins', street: '423 Earls Road', img: 'men/3.jpg' }
    ];

    // Wait until both modules are initialised to select the initial user
    $timeout(function(){
        $scope.select( $scope.users[ state.moduleConfig.selectedUserIndex ] );
    });

    $scope.select = function( user ) {
        //Highlight the selected user in the list
        selectedUser.isSelected = false;
        user.isSelected = true;
        selectedUser = user;

        //Notify other modules of the selection
        eventHub.trigger( 'userSelected', user );
    };
}])
.value( 'eventHub', eventHubBetweenModules );

The details panel

angular.module('userdetails', [] )
.controller('userdetailsController', 
    [ '$scope', 'eventHub', function( $scope, eventHub ) {
    $scope.user = null;

    eventHub.on( 'userSelected', function( event, user ){
        $scope.user = user;
        $scope.$apply();
    });
}])
.value( 'eventHub', eventHubBetweenModules );

Bootstrapping

So far, so good. You’ve probably noticed the code that selects the initial user based on a configured state

$timeout(function(){
    $scope.select( $scope.users[ state.moduleConfig.selectedUserIndex ] );
});

This parameter is part of the GoldenLayout configuration. Since the bootstrapping process for modules is pretty much always the same we'll write one generic component for it and tell it via configuration which template to load and which module to create

componentName: 'angularModule',
componentState: {
    module: 'userlist',
    templateId: 'userlistTemplate',
    moduleConfig: { selectedUserIndex: 2 }
}

Here's the whole GoldenLayout initialisation

var myLayout = new GoldenLayout({
    content:[{
        type: 'row',
        content: [{
            // Userlist's width is 33% of the total width
            width: 33,
            title: 'Registered Users',
            type: 'component',
            componentName: 'angularModule',
            componentState: {
                module: 'userlist',
                templateId: 'userlistTemplate',
                moduleConfig: { selectedUserIndex: 2 }
            }
        },{
            type: 'component',
            title: 'Selected User',
            componentName: 'angularModule',
            componentState: {
                module: 'userdetails',
                templateId: 'userDetailTemplate'
            }
        }]
    }]
});

myLayout.registerComponent( 'angularModule', AngularModuleComponent ); myLayout.init();

AngularModuleComponent

The AngularModuleComponent is the glue between Angular and GoldenLayout

var AngularModuleComponent = function( container, state ) {
    // Templates are specified as template tags in the DOM.
    var html = $( '#' + state.templateId ).html(),
        element = container.getElement();

    // Populate the element first
    element.html( html );

    // Expose container and state to the module. If multiple instances of
    // the same module are created this will override the previous modules container
    // and state with the current (correct) one
    angular
        .module( state.module )
        .value( 'container', container )
        .value( 'state', state );

    // Actually kick off Angular's magic
    angular.bootstrap( element[ 0 ], [ state.module ] );
};
